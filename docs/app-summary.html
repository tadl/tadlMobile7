TADL Mobile

A native-feeling mobile app for Traverse Area District Library cardholders, built with Ionic, Angular, and Capacitor.

Overview

TADL Mobile gives patrons quick access to account tools, catalog discovery, library news, events, locations, and utility features from a single mobile app experience.

The app is designed around practical patron tasks: searching the catalog, placing and managing holds, checking due dates, browsing events, viewing service alerts, and getting to the right branch information fast.

Audience

  • TADL cardholders using iPhone or Android
  • Patrons who want faster account access than the mobile web
  • Staff supporting patrons with account, hold, and location questions

Core Features

  • Account access for checkouts, holds, fines, lists, card display, and preferences
  • Catalog search with filters, ISBN barcode scan, MeLCat shortcut, and item detail modals
  • Hold placement and management, including suspend, cancel, and pickup-location changes
  • Event browsing with detailed views and “Add to calendar” support
  • Newsletter feed, webcams page, and branch/location detail screens

Data Sources

  • Aspen Discovery APIs for search, holds, checkouts, and grouped work details
  • TADL feed proxies for newsletters and webcams
  • Locations JSON with service exceptions and branch metadata
  • Cached local data for faster repeat loads and smoother navigation

Recent Enhancements

  • Newsletter page moved to feed-backed content with richer detail rendering
  • Webcams page moved to feed-backed entries and safer external playback
  • Location service alerts now support Eastern-time same-day and next-day exceptions
  • Homepage now surfaces upcoming branch closures and modified hours
  • Item detail views now support calendar export, stronger list feedback, and better hold-state handling
  • Preferences page was clarified with link-mode and theme explanations
  • Small-screen layout issues were addressed on Account and Location detail pages
  • Contrast, spacing, and corner-radius polish improved readability and consistency

Technical Summary

The app uses Ionic Angular for the UI, Capacitor for native packaging and device features, and a mix of live API calls plus local caching to keep the interface responsive. It is optimized for both logged-in patron workflows and quick public browsing tasks.

Current release focus: polish, performance, and dependable real-world patron flows rather than major new feature development.
